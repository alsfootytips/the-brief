# The Brief

A weekly investing brief designed to teach how to read markets and understand why
stocks move, rather than tell anyone what to buy.

## Files

```
The Brief/
├── the-brief.html             archive shell — open this in a browser
├── issue-2-reference.html     design reference (do not edit)
├── issues/
│   ├── issue-002.js           Issue No. 2 content data
│   └── issue-003.js           empty template (draft: true; hidden until ready)
├── watchlist.json             tickers the pipeline tracks
├── update.py                  ingestion pipeline (Phase 2)
├── requirements.txt           Python deps
├── .env.example               copy to .env and fill in keys
└── data/                      generated by update.py (committed)
    ├── live.js
    └── movers.js
```

## Conventions

- No build step. Single self-contained HTML files. Vanilla JS only.
- Fonts: Fraunces, JetBrains Mono, Inter via Google Fonts.
- Every claim tagged by confidence: Fact / Interp / Spec.
- No stock picks — frameworks only.

## Running the pipeline locally

One-time setup:

```sh
cd "/Users/alexvinning/Documents/The Brief"
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Then open `.env` and fill in at least:

- `FINNHUB_API_KEY` — free at finnhub.io (60 calls/minute is plenty)
- `SEC_USER_AGENT` — any string with your email; SEC requires identification

Run an ingest:

```sh
source venv/bin/activate
python update.py
```

That writes `data/movers.js` and `data/live.js`. Refresh `the-brief.html` in your browser to see them in the Live Feed and Movers Board views.

## What runs in each phase

- **Phase 1** (done): archive shell + Issue No. 2 ported into data.
- **Phase 2** (done): manual ingestion via `python update.py`.
- **Phase 3** (infrastructure done, awaiting API key): Claude API layer that summarizes raw ingest into the Brief's voice with Fact/Interp/Spec tags. Adds a daily entry into the current week's issue.
- **Phase 4**: Pushover notifications on trigger events.
- **Phase 5** (in progress): GitHub Actions cron + GitHub Pages for unattended daily runs and phone access.

## Deployment (Phase 5)

GitHub Actions workflow at `.github/workflows/update.yml` runs the pipeline twice on weekdays (07:00 ET pre-market, 17:00 ET post-close), commits data updates back to the repo. GitHub Pages serves `the-brief.html` and all data files from the same branch.

Secrets to add in the GitHub repo (Settings → Secrets and variables → Actions):
- `FINNHUB_API_KEY` (required)
- `SEC_USER_AGENT` (required; format: "Name email@example.com")
- `ANTHROPIC_API_KEY` (optional, needed for Phase 3 editorial entries)
- `FRED_API_KEY`, `PUSHOVER_USER_KEY`, `PUSHOVER_APP_TOKEN` (optional, future phases)

Kill switch: create a file named `PAUSED` at repo root to skip all scheduled runs.
